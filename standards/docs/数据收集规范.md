# 文创指南数据收集规范

## 1. 数据分类与定义

### 1.1 品牌案例
**定义:** 具有明确品牌名称、市场化运营的文创实体

**收录标准:**
- ✅ 真实存在的品牌/公司/机构
- ✅ 有明确的产品或服务
- ✅ 有公开的市场信息（官网、电商店铺、媒体报道）
- ✅ 品牌名称具体，非泛称

**示例:**
- ✅ 故宫淘宝、泡泡玛特、观夏、茶颜悦色
- ❌ 北京文创品牌、上海文旅综合体（过于泛化）

### 1.2 产品案例
**定义:** 具体的文创产品单品或系列

**收录标准:**
- ✅ 具体产品名称
- ✅ 明确的价格区间
- ✅ 清晰的产品图片（3-5张）
- ✅ 销售渠道信息

**示例:**
- ✅ 故宫日历2026书画版、泡泡玛特Molly系列
- ❌ 某品牌文具系列（过于笼统）

### 1.3 政策文件
**定义:** 政府部门发布的正式政策文件

**收录标准:**
- ✅ 真实存在的政策文件
- ✅ 有明确的发布机构和文号
- ✅ 可查证的发布日期
- ✅ 政策内容具体，非泛泛而谈

**示例:**
- ✅ 《关于促进文化创意产业发展的若干意见》（文号：XXX〔2024〕XX号）
- ❌ 某市将大力发展文创产业（非正式文件）

## 2. 数据质量标准

### 2.1 必填字段

#### 品牌案例
| 字段 | 要求 | 示例 |
|------|------|------|
| name | 品牌全称 | 故宫淘宝 |
| region | 所在省份 | 北京市 |
| city | 所在城市 | 东城区 |
| category | 细分品类 | 博物馆文创 |
| products | 主要产品 | 彩妆、文具、饰品 |
| feature | 核心特色 | 宫廷风设计 |
| type | 数据类型 | real_brand |

#### 产品案例
| 字段 | 要求 | 示例 |
|------|------|------|
| name | 产品全称 | 故宫日历2026书画版 |
| brand | 所属品牌 | 故宫淘宝 |
| price | 价格 | ¥126 |
| category | 产品类别 | 日历/文具 |
| images | 图片URL数组 | ["url1", "url2"] |
| channels | 销售渠道 | 天猫、小程序 |
| type | 数据类型 | real_product |

#### 政策文件
| 字段 | 要求 | 示例 |
|------|------|------|
| title | 政策全称 | 关于促进文创产业发展的实施意见 |
| region | 发布地区 | 北京市 |
| date | 发布日期 | 2025-01-15 |
| doc_number | 文号 | 京政发〔2025〕3号 |
| source_url | 来源链接 | https://www.beijing.gov.cn/... |
| type | 数据类型 | real_policy |

### 2.2 数据来源分级

| 级别 | 可信度 | 来源 | 使用建议 |
|------|--------|------|----------|
| A | 95%+ | 官方网站、财报、政府文件 | 优先使用 |
| B | 80-95% | 主流媒体报道、电商平台 | 可使用 |
| C | 60-80% | 社交媒体、行业分析 | 需交叉验证 |
| D | <60% | 估算、推测 | 标注来源，谨慎使用 |

## 3. 禁止收录的数据

### 3.1 品牌案例
- ❌ 模板生成的泛化名称（如"{城市}文创品牌"）
- ❌ 不存在的虚构品牌
- ❌ 个人工作室（无工商注册）
- ❌ 已倒闭/注销的品牌（除非做历史研究）

### 3.2 产品案例
- ❌ 概念产品（未上市）
- ❌ 侵权/山寨产品
- ❌ 无明确价格的产品
- ❌ 图片无法验证的产品

### 3.3 政策文件
- ❌ 非正式文件（领导讲话、会议摘要）
- ❌ 已过期的政策（除非做历史研究）
- ❌ 无法查证来源的文件
- ❌ 地方性政策但无具体文号

## 4. 数据收集流程

### 4.1 品牌案例收集

```
Step 1: 确定目标品牌
  - 从官方渠道获取品牌列表
  - 优先选择知名度高、数据公开的品牌

Step 2: 信息采集
  - 官网: 品牌介绍、产品信息
  - 电商: 天猫/京东店铺数据
  - 媒体: 新闻报道、行业分析
  - 社交: 小红书、抖音官方账号

Step 3: 数据验证
  - 交叉验证多个来源
  - 确认品牌真实存在
  - 核实关键数据（成立时间、销售额等）

Step 4: 录入系统
  - 按规范填写字段
  - 上传图片（3-5张）
  - 标注数据来源
```

### 4.2 产品案例收集

```
Step 1: 选择代表性产品
  - 销量高、知名度高
  - 能代表品牌特色
  - 有完整的产品信息

Step 2: 产品拍摄/收集图片
  - 产品主图（白底）
  - 细节图（材质、工艺）
  - 场景图（使用场景）
  - 包装图（品牌体验）

Step 3: 信息整理
  - 产品名称、规格、价格
  - 销售渠道、销量数据
  - 用户评价、市场反馈

Step 4: 质量检查
  - 图片清晰度
  - 信息完整性
  - 数据准确性
```

### 4.3 政策文件收集

```
Step 1: 确定政策来源
  - 政府官网（首选）
  - 官方公报
  - 权威媒体转载

Step 2: 下载原始文件
  - PDF版本（如有）
  - 网页版本
  - 保存截图备份

Step 3: 提取关键信息
  - 标题、文号、发布日期
  - 发布机构、适用范围
  - 核心内容摘要

Step 4: 验证真实性
  - 核对文号格式
  - 确认发布机构
  - 检查官方来源
```

## 5. 数据更新与维护

### 5.1 定期更新
- **品牌案例:** 每季度更新一次数据
- **产品案例:** 每月更新销量、评价数据
- **政策文件:** 实时跟踪新发布政策

### 5.2 数据清理
- 每半年清理一次过期数据
- 删除已倒闭品牌（移至历史库）
- 更新失效链接

### 5.3 质量监控
- 每月抽查10%数据
- 检查数据准确性
- 处理用户反馈

---

## 6. 违规处理

### 6.1 数据质量问题
| 级别 | 问题 | 处理 |
|------|------|------|
| 轻微 | 字段缺失、格式错误 | 补充完善 |
| 中等 | 数据过时、链接失效 | 更新或删除 |
| 严重 | 虚假信息、侵权内容 | 立即删除 |

### 6.2 收集人员责任
- 对收集的数据质量负责
- 必须验证数据来源
- 不得编造或抄袭数据

## 7. 附录

### 7.1 推荐数据源

**品牌案例:**
- 天猫/京东品牌旗舰店
- 品牌官方网站
- 36氪、虎嗅等科技媒体
- 小红书、抖音官方账号

**产品案例:**
- 电商平台产品页
- 品牌官方商城
- 用户评价数据
- 行业报告

**政策文件:**
- 国务院政策文件库
- 各省市政府官网
- 文化和旅游部官网
- 国家文物局官网

### 7.2 工具推荐
- 数据抓取: Python + Scrapy
- 图片处理: Photoshop、Figma
- 数据验证: 交叉比对多个来源
- 版本控制: Git + GitHub
